# 项目总结：见习侦探的暑期事件簿

## 缘起

我有一个刚读完初一、即将升初二的孩子。暑假 50 天，学校布置了一大堆作业——古诗文背诵、英语单词、数学 17 套练习、物理预习、语文红色经典阅读、文创设计、历史报纸特刊、道法思维导图、体育每日锻炼……分散在各科 PDF 里，看着就头大。

孩子需要管理，我需要监督，但谁也不喜欢每天对着一个枯燥的清单勾勾画画。

于是有了这个想法：把整个暑假学习做成一个侦探游戏。

## 做了什么

### 从"打卡"到"破案"

传统的打卡应用，完成一个任务就勾掉一行。我们把它改成了——完成学习任务 → 随机掉落线索 → 集齐线索 → 自己推理破案。

这不是换个名字而已。孩子收集的不是"积分"，而是具体的、有内容的**物证**（比如"作业本最后一页有半枚蓝色墨水指纹，指纹纹路较浅，像是孩子的手"），这些物证需要他看完、理解、串联，才能回答推理问题。50 个案件，从 2 条线索的入门案到 6 条线索的终极案，每个都有完整的案情、线索、问题、选项和破案解释。

这背后的工作量：为 50 个案件撰写了 200+ 条线索文本、50 道推理问题、150 个选项、50 条破案解释。每条线索都要求逻辑自洽、结论可推导、不能太简单也不能太难。这对一个初一的孩子刚刚好。

### 从"系统推送"到"主动抽签"

原本设计是系统每天自动显示一条金句。但我意识到——推送的东西不值钱，自己抽到的才珍惜。

于是改成了抽签筒：孩子每天自己点抽签按钮，从 75 条金句里随机抽一条。75 条抽完才重置，不会连续重复。加上可以选每日心情，这些数据都叠在日历上——一个暑假下来，哪天开心、哪天疲惫，一目了然。

### 从"猜测的单词表"到"逐字校对"

最初英语单词是我根据记忆写的人教版词汇，后来发现教材是沪教版，完全对不上。于是走了一条很曲折的路：

1. 先尝试 OCR 识别 PDF，中文乱码严重
2. 改用 600 DPI 高清图 + 中英文混合识别，`pdftotext` + `tesseract` 双引擎
3. 用户提供了准确的 txt 版本，我写了 Node 解析脚本逐行提取
4. 最终版本：七下 8 个单元 270+ 词、八上 8 个单元 270+ 词，全部来自真实教材

这件事的教训：**数据准确度是学习工具的生命线**，不能靠猜。

### 从"代码能跑"到"孩子能用"

写代码和做产品是两回事。这个项目经历了大量"看起来没问题，用起来不舒服"的调整：

- 打卡后页面跳回顶部 → 双层 rAF 恢复滚动位置 → 最终改为即时刷新去掉 300ms 延迟
- 暑假作业展开后页面太长 → 改成按学科折叠
- 心情趋势占据大片空白 → 从独立区域 → 合并到日历格子上
- 英文课文太长不适合背诵 → 全部替换为单词表
- 档案页显示 `ch_sx_0_1` 而不是"三角形的概念" → 写了全类型 ID 解析器
- 深色界面不适合女孩子 → 换成暖粉 + 淡紫渐变
- 情绪 emoji 太小看不清 → 从圆圈改为胶囊按钮带文字
- 学科进度横条太单调 → SVG 技能环 + 光晕动画
- "重置全部数据"可能被误触 → 删掉

这些改动没有一个涉及核心算法，但每一个都直接影响使用体验。

### 从"一个人用"到"可以让别人用"

最终产物是一个纯静态网站——`dist/` 目录里三个文件，丢到任何静态托管服务就能用。数据存在浏览器里，不需要账号、不需要服务器、不需要数据库。这在技术上是一个刻意的设计选择：让部署成本为零，让分享门槛为零。

## 技术选型与取舍

| 决策 | 选择 | 原因 |
|------|------|------|
| 框架 | 不用，纯原生 JS | 项目规模不到一万行，引入框架是过度设计 |
| 构建 | Vite | 模块化开发 + 一键打包，零配置 |
| 存储 | IndexedDB | 浏览器原生，离线可用，不需要后端 |
| UI 库 | 不用，纯 CSS | 保持零依赖，样式完全可控 |
| 移动端 | iPad 480px 优先 | 孩子的使用场景就是 iPad |
| 桌面端 | 侧边栏布局 | 家长在电脑上查看更方便 |

## 个人感受

这是我目前做的第一个真正"有用"的项目——不是 demo，不是练习，是一个孩子明天就可以打开用的东西。

作为人力资源从业者，我习惯做流程、做管理、做跟办。把一个管理思维转化成让孩子兴奋的游戏化体验，这中间的思考和尝试，比最终的 9000 行代码更有价值。

**最好的部分是**：孩子参与了这个过程。她画了自己的头像上传到页面里；她选了心情；她会说"英文的背诵是单词不是课文"——这句话推动了英语内容的全面重构。她不是被动的使用者，她是这个产品的共同创作者。

**最意外的是**：我原以为最花时间的是写代码，结果代码反而是最快的部分。最花时间的是：理解教材内容、整理暑假作业清单、为 50 个案件编写有趣的破案故事、反复调整 UI 直到孩子说"可以了"。
