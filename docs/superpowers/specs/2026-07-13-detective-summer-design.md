## 一、项目概述

- **名称**：见习侦探的暑期事件簿（L's Case Journal）
- **定位**：侦探主题暑假学习打卡应用，初一升初二学生
- **设备**：iPad（移动端优先，max-width 680px），桌面 Chrome 辅助
- **周期**：2026.07.13 ~ 2026.08.31（50天）
- **特殊日**：周日全天休息、旅游期（8月初约10天）仅背诵打卡
- **技术栈**：Vite + 原生 JavaScript ES Modules + 纯 CSS，零运行时依赖
- **数据存储**：IndexedDB（主存储）+ JSON 文件导出备份

```
│   │   ├── progress.js        # 打卡/星星/连续天数/完成率/等级计算
│   │   └── blindbox.js        # 盲盒解锁/每日盲盒/阶梯线索逻辑
│   ├── ui/
│   │   ├── header.js          # 顶部导航栏渲染
│   │   ├── tab-search.js      # Tab1 今日搜证

│   │   └── sounds.js          # Web Audio API 音效生成
│   └── styles/
│       └── main.css           # 全局样式
```

### 3.2 模块依赖关系

```
main.js（入口：初始化、Tab 切换、全局状态）
  ├── config.js          ← 常量配置，被所有模块引用
  ├── data/
  │   ├── db.js          ← 唯一的 IndexedDB 读写入口
  │   ├── tasks.js       ← 任务定义数据（静态）
  │   └── blindboxes.js  ← 盲盒内容数据（静态）
  ├── core/
  │   ├── engine.js      ← 日期工具、艾宾浩斯间隔
  │   ├── progress.js    ← 打卡/星星/等级逻辑
  │   └── blindbox.js    ← 盲盒/阶梯线索逻辑
  ├── ui/                ← 渲染层，不直接操作 db.js
  ├── audio/sounds.js    ← 音效层
  └── styles/main.css
```

### 3.3 数据流（单向）

```
用户点击完成
  → UI 层调用 core/progress.js 的 completeTask(state, id)
    → progress.js 计算新状态（status, nextReviewDate, stars）
      → progress.js 调用 data/db.js 的 saveState(state)
        → db.js 写入 IndexedDB
      → progress.js 返回更新后的 state + 事件（starsEarned, badgeUnlocked 等）
    → UI 层拿到新 state，重新渲染
    → audio/sounds.js 播放音效（根据事件类型）
```

## 四、数据存储

### 4.1 IndexedDB 设计

数据库名：`detective_summer_v2`

**表：state**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | string | 固定为 `"current"`，单条记录 |
| tasks | object | `{ [id]: { status, learnedDate, reviewCount, nextReviewDate } }` |
| activeDays | array | 已打卡日期列表 |
| customTasks | object | `{ [date]: [{ desc, done }] }` |
| dailyDone | object | `{ "daily_<id>_<date>": true/false }` |
| chapterDone | object | `{ "ch_<sub>_<chapterId>_<date>": true/false }` |
| chapterDone | object | `{ "ch_<sub>_<chapterId>_<date>": true/false }` |
| sportDone | object | `{ "sport_<sportId>_<date>": true/false }` |
| blindboxes | object | 已解锁结案盲盒记录 |
| dailyBlindboxes | object | 已弹出每日盲盒记录 |
| diary | object | `{ [date]: { text, stars } }` |
| stars | object | `{ [date]: number }` 每日星星数 |
| badges | array | 已解锁成就列表 |
| _version | number | 数据版本号，初始 1 |

**表：backups**

| 字段 | 类型 | 说明 |
|------|------|------|
| id | auto | 自增 |
| timestamp | number | 备份时间戳 |
| data | object | 完整 state 快照 |

### 4.2 备份策略

- 每次数据变更时自动在 IndexedDB 的 backups 表存一份快照（保留最近 7 天）
- Tab3 提供手动导出：下载 JSON 文件到本地
- Tab3 提供手动恢复：选择 JSON 文件导入，校验格式后覆盖

## 五、任务体系

### 5.1 背诵任务（语文27项 + 英语16项 = 43项）

- 语文：八年级上必背古诗文（三峡、答谢中书书、记承天寺夜游等 27 篇）
- 英语：U1-U8 单词（每单元拆 2 批，共 16 项）
- 分配规则：每 2 个学习日分配 1 篇新背诵（语文英语交替均匀分布）
- 非学习日（周日、旅游期）自动跳过
- 触发艾宾浩斯复习（间隔：1天 → 2天 → 4天 → 7天 → 15天，共5次）
- 每日复习上限 10 项，超出排队

### 5.2 每日固定任务

| 科目 | 任务 | 周六变化 |
|------|------|---------|
| 语文 | 阅读理解1篇、推荐书目阅读 | -- |
| 英语 | 时文阅读、阅读理解1篇 | 阅读理解 → 作文+听力 |

**渲染规则**：
- 每天自动出现在 Tab1 对应科目面板中，无需预分配 suggestedDate
- 与背诵任务混合展示在同一科目折叠面板下，用灰色"每日"标签区分
- 每个学习日（非周日）自动可操作，不需等到某天"解锁"
- 周日/旅游期按规则显示或隐藏（周日全部隐藏，旅游期仅保留英语每日任务）

**不参与**：
- 不参与艾宾浩斯复习
- 不预分配日期
- 不计入总进度百分比
- 计入当日星星（每项 +1 星）

### 5.3 数学/物理：章节选择（v2 改进）

**原设计**：自由输入框，孩子自己打字写学了哪一节
**问题**：打字麻烦、容易忘记、体验不一致
**新设计**：改为章节列表勾选

```
📐 密码解析（数学）
┌─────────────────────────────────────┐
│ 第十三章 三角形                 [▸]  │
│   13.1 三角形的概念         ✅ 07-15 │
│   13.2 与三角形有关的线段       ○    │
│   13.3 三角形的内角与外角       ○    │
│                                     │
│ 第十四章 全等三角形           [▸]    │
│   14.1 全等三角形及其性质       ○    │
│   ...                              │
└─────────────────────────────────────┘
```

- 章节默认折叠，点击展开/收起
- 每个小节前有圆形勾选框，点击完成
- 已完成显示绿色勾 + 日期
- 每完成一节，弹出"线索收集 +1"微动画
- 章节进度条显示"本章 N/M 节已完成"

**数学章节**（19节）：第十三章 三角形(3) / 第十四章 全等三角形(3) / 第十五章 轴对称(3) / 第十六章 整式的乘法(3) / 第十七章 因式分解(2) / 第十八章 分式(5)

**物理章节**（29节）：第1章 机械运动(4) / 第2章 声现象(5) / 第3章 物态变化(5) / 第4章 光现象(5) / 第5章 透镜及其应用(5) / 第6章 质量与密度(4)

### 5.4 体育：选项勾选（v2 新增）

每天可选择完成的运动项目：

| 项目 | 图标 |
|------|:--:|
| 游泳 | 🏊 |
| 拉伸 | 🧘 |
| 跑步 | 🏃 |
| 跳绳 | 🪢 |

- 每天四个选项独立勾选，不限数量
- 点选即完成，和章节选择交互一致
- 每完成一项 +1 星
- 不参与艾宾浩斯复习
- 旅游期正常显示（运动不受旅游影响）

### 5.5 特别行动（自定义）

- 学生自行添加（如"洗碗""整理房间"）
- 每天最多 5 项
- 不计入背诵复习、不计入理科进度
- 计入打卡天数

## 六、三层激励体系（v2 核心）

### 第一层：每日盲盒（见面奖励）

- 每天首次打开页面自动弹出
- 零门槛，让孩子愿意每天打开
- 内容：随机逻辑推理题 / 侦探冷知识
- 来源：`data/blindboxes.js` 的 A、B 库

### 第二层：星星阶梯（努力奖励）

**赚星规则**：

| 行为 | 星星 |
|------|:----:|
| 完成一项背诵任务（新学） | +1 |
| 完成一项复习任务 | +1 |
| 完成一项每日固定任务 | +1 |
| 完成一节数学/物理 | +1 |
| 完成一项体育运动 | +1 |
| 完成一项特别行动 | +1 |
| 写学习日记 | +1 |
| 完成每日挑战 | +3 |
| CASE CLOSED（全清） | +5 |

**阶梯奖励**：

| 当日累计星星 | 解锁 |
|:-----------:|------|
| 3 星 | 铜质线索：额外一条侦探冷知识 |
| 5 星 | 银质线索：触发隐藏成就进度 |
| 8 星 | 金质线索：稀有盲盒（侦探小说推荐 + 激励语） |
| 全清 + 10 星以上 | 名侦探宝箱：当日星星翻倍（上限 20） |

**周趋势**：Tab3 显示本周星星柱状图，和上周对比

### 第三层：结案盲盒（全清奖励）

- 当日所有可见任务全部完成 → CASE CLOSED 红色印章动画
- 解锁当日结案盲盒
- 门槛最高，奖励最重

## 十四、节假日与边界

- 周日全天休息（显示侦探休息页 + 本周统计）
- 旅游期（8/1-8/10）：数学物理隐藏，仅背诵 + 每日固定
- 周六正常：英语阅读理解自动替换为作文+听力
- 不可对未来日期打卡
- 不可早于 startDate
